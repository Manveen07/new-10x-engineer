# Monthly Execution Checklist

This repo now follows a monthly plan. Existing daily exercise folders remain useful drills, but the canonical flow is month-by-month.

## Month 1 - Evals Foundations (self-paced, free path)

- [ ] Set up or update portfolio site (Quarto or Astro, live URL by end of Week 1).
- [ ] Read Hamel `evals-faq` end-to-end.
- [ ] Read Eugene Yan "Patterns for Building LLM-based Systems & Products."
- [ ] Read Hamel `llm-judge` post.
- [ ] Pull 50 real classifier traces from past Clay/winery/HVAC runs.
- [ ] Open-code 50 traces using the Three Gulfs framework.
- [ ] Convert open-coding notes into an axial failure taxonomy.
- [ ] Build a lightweight annotation viewer (FastHTML or Streamlit).
- [ ] Calibrate an LLM-as-judge with critique shadowing on a sample, target >90% agreement.
- [ ] Draft Project 1 design doc.
- [ ] Publish post 1.
- [ ] Update [PROGRESS.md](./PROGRESS.md) every Sunday.

## Month 2 - Business Classifier Ship

- [ ] Watch DLAI *Improving Accuracy of LLM Applications* (free, 1-2 hrs).
- [ ] Define Pydantic output schemas.
- [ ] Add Instructor-based structured outputs.
- [ ] Add retrieval/evidence collection path.
- [ ] Add Langfuse instrumentation from the start.
- [ ] Hand-label 100-company golden dataset.
- [ ] Build binary judge checks per dimension.
- [ ] Iterate judge prompt to more than 90% label agreement.
- [ ] Add confusion matrix, cost, and latency sections.
- [ ] Deploy Project 1.
- [ ] Publish post 2.

## Monthly Self-Review

Answer these on the last Sunday of every month:

- Did I ship something this month that did not exist last month?
- Can I explain one new concept in 60 seconds?
- Is my GitHub commit graph visible?
- Is my blog still updated?
- Did I attend or watch one community event?
- What did I cut from the plan, and was the cut honest?
- If a hiring manager opened my portfolio today, would eval results be visible quickly?
