## Standing rules (re-read every Sunday for first 3 months)

1. **Eval before scale.** Every project gets a golden dataset before optimization.
2. **Workflow before agent.** Default to deterministic code; agentic dynamism only at the node where it's genuinely needed.
3. **One framework or none.** Raw SDK + Pydantic + Instructor is the default.
4. **Production thinking shows up in the README.** "Where it fails," cost-per-call, latency p50/p95, eval table.
5. **Public > private.** Ship to a URL. Write the post. Tag people.
6. **Lean into your wedge.** GTM domain + Clay community + production LLM experience is your unfair advantage.

Anti-patterns to refuse: tutorial-finishing, framework collecting, fake claims, demo without evals, hype-posting on LinkedIn. Full list in [MASTER_PLAN.md](./MASTER_PLAN.md) Section 7.

## When something breaks

- **Stuck on tooling:** check [MASTER_PLAN.md](./MASTER_PLAN.md) Day 0 Prerequisites — likely something there.
- **Behind on the week:** read the "Behind if" markers in the current week file. Recover the must-have item before starting next week.
- **Two Sundays missed in a row:** that's the canary. Reset on the next Sunday with a smaller, honest plan. Don't pretend.
- **Lost or overwhelmed:** close every file except this one. Open the current week file. Pick the easiest task in the Daily Plan. Set a timer. Start.
